## 1. 전체 구조

**IA**

고객별고정가 관리

- 고객별고정가 관리
    - 고객별고정가 상세
- 상품별고정가 관리

이 문서는 **3개의 화면**을 다룬다.

- 2절 — 고객별고정가 관리: 고객 전체 목록 조회·검색, 고객 추가
- 3절 (2절의 하위기능) — 고객별고정가 상세: 특정 고객에 묶인 상품 및 고정가 등록·수정·삭제
- 4절 — 상품별고정가 관리: 상품 기준으로 전체 고정가 조회·수정

2,4절 목록은 **같은 데이터**를 바라보지만 **진입 축(고객 vs. 상품)**이 다르다.

**유효성 — 정가 or 판매가를 초과하는 고객별고정가 입력 차단**

- 고객별고정가는 해당 상품의 **정가, 판매가보다 크게 입력할 수 없다.** 초과 입력 시 저장을 거부하고 에러 메시지를 표시한다.
- 근거: 기준지침 §3 예외사항 — 정가보다 캠페인할인가가 높으면 캠페인할인가가 무효화되고 정가로 판매되는 시스템 구조이므로 (고객이 계약가가 아닌 정가로 결제할 수 있음), BO 입력 단계에서 사전 차단한다.

**AI쿠폰금액 노출 규칙:** AI기본쿠폰과 AI개인화쿠폰은 상호 배타적으로 하나만 적용된다. AI개인화쿠폰 금액이 있으면 개인화쿠폰만, 없으면 기본쿠폰만 노출하고 나머지는 "-"로 표시한다.

## 5. 일괄 업로드/다운로드 (US-02) — 개발 검토필요

- **진입점 2곳:**
    - 고객 목록 상단 > [엑셀 다운로드], [일괄 업로드] 버튼 클릭
    - 상품 목록 상단 > [엑셀 다운로드], [일괄 업로드] 버튼 클릭

> ***\*\*\* 고정가관리에 기 등록되지 않은 고객은 일괄 업로드 불가***  
> ***\*\*\* 고정가관리에 기 등록된 고객에 대하여 신규 상품 추가는 가능함***

고객별고정가에 이미 등록된 고객만 업로드 대상으로 허용한다.  
미등록 고객에 대한 행은 그룹 3 시스템 대조 단계에서 회원번호 미존재 오류로 차단된다.

- **업로드 시 업데이트 대상 항목 (2개):** 고객별고정가, 기타 — 이외 항목은 수정 불가
- **key 식별자:** 회원번호, 상품ID
- **업로드 방식:** 오류 발견 즉시 나머지 행 검증 중단
- **엑셀양식 컬럼:** 상품별고정가 관리 테이블 column과 동일함 (4-1 목록 컬럼 참조)
    - 특정 컬럼은 아래와 같이 헤더명 표현
        - `회원번호(필수)`
        - `상품ID(필수)`
        - `고객별고정가(필수)`

#### 그룹 0 | 파일 사전 검증 — 파싱 전, 즉시 차단

| 케이스 | 에러 문구 |
| --- | --- |
| 잘못된 확장자 | `업로드할 수 없는 파일 형식입니다 (.xlsx, .csv만 허용)` |
| 파일 용량 초과 | `파일 용량이 초과되었습니다 (최대 10MB)` |
| 행 수 초과 | `업로드 가능한 최대 행 수를 초과했습니다 (최대 5,000건)` |

#### 그룹 1 | 형식 오류 — 파일 파싱 즉시 체크, DB 불필요

| 케이스 | 에러 문구 |
| --- | --- |
| 필수값 누락 | `n행: 필수항목(회원번호,상품ID,고객별고정가)가 누락되었습니다` |
| 고정가 형식 오류 | `n행: 고정가는 0보다 큰 숫자로 입력해주세요` |

#### 그룹 2 | 파일 내 중복 — 전체 행 파싱 완료 후 한 번에 체크

| 케이스 | 에러 문구 |
| --- | --- |
| 동일 (회원번호+상품ID) 중복 | `(N행, M행) 동일한 회원번호x상품이 중복되어 있습니다` |

#### 그룹 3 | 시스템 대조 — DB 배치 조회 한 번으로 묶어서 처리

| 케이스 | 에러 문구 |
| --- | --- |
| 회원번호 미존재 | `N행: 고객별고정가관리에 등록되지 않은 회원번호입니다 (회원번호: {회원번호 입력값})` |
| 상품ID/ERP코드 미존재 | `N행: 존재하지 않는 상품입니다 (상품ID: {상품ID 입력값})` |
| 고정가 > 판매가 or 정가 | `N행: 고정가(X원)가 판매가(X원)를 초과합니다` / `N행: 고정가(X원)가 정가(X원)를 초과합니다` |
