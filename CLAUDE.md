### 엑셀 일괄 업로드
- **업로드 방식:** 고객 단위 롤백 — 한 고객의 행 중 하나라도 오류면 해당 고객 전체 롤백, 오류 없는 고객은 정상 등록 (부분 성공 가능)
- **검증 방식:** Fail-fast — 오류 발견 즉시 나머지 행 검증 중단
- **업로드 대상:** 고객별고정가에 기 등록된 고객만 가능 (신규 상품 추가는 가능)
- **업데이트 가능 항목:** 고객별고정가, 기타 (2개만)
- **key 식별자:** 회원번호 + 상품ID

**검증 그룹**
| 그룹 | 내용 | 처리 시점 |
| --- | --- | --- |
| 0 | 파일 사전 검증 (확장자, 용량 10MB, 행수 5000건) | 업로드 즉시 |
| 1 | 형식 오류 (필수값 누락, 고정가 형식) | 파일 파싱 시 |
| 2 | 파일 내 중복 (동일 회원번호+상품ID) | 전체 파싱 후 |
| 3 | 시스템 대조 (회원번호·상품ID 존재여부, 고정가>판매가/정가) | DB 배치 조회 |

## 차별화상회 정책서 요약 (관련 정책 발췌, 2026-07-06 학습)

### 기존 가격 노출 우선순위 (상품목록·상품상세 공통, OP202310 기준)
고객별고정가가 없는 상품은 아래 우선순위를 따른다.
1. 타임세일 최고 할인율 (슈퍼회원 등급 영향 없음)
2. 일반 캠페인 최고 할인율 (슈퍼회원이면 10%와 비교)
3. 가장 최근 등록 캠페인
4. 정가

**OP202606 override 관계**: 이탈고객 + 유효 고객별고정가 → 위 우선순위 전체를 override, 단일 금액만 노출.

### 가격 계산식
- `판매가 = round(정가 - (정가 * 할인율/100))`  (소수점 첫째 자리 반올림)
- `단위당 가격 = ceil(판매가 / (낱개규격 * 입수))`

### 캠페인 중복 시 노출 기준 (OP202307)
1순위) 할인율 높음 → 최종 판매가 낮은 순 / 2순위) 최근 시작한 순 / 3순위) 캠페인 등록일시

### 묶음상품 정책 (OP202502, 2026.03.20 추가)
- 모든 상품 목록에 낱개/묶음 상품 개별 노출
- 묶음상품 가격 노출 = 할인판매가(낱개 기준) × 묶음수량
- 고객별고정가도 낱개 기준 입력 → 묶음 노출 시 `고정가 × 묶음수량`. 기존 가격 입력 방식과 동일하므로 별도 처리 불필요.
- **검토 중**: 상품상세 옵션 목록에서 낱개엔 고정가 O·묶음엔 고정가 X 혼재 케이스 (customer.md OI-6)

### 쿠폰 정책 (관련 부분)
- 장바구니쿠폰: 주문당 최대 1개, 정액할인만 지원
- 쿠폰 계산 순서: 총 상품금액 → 쿠폰할인 → 배송비 → 포인트 → 잔여금액 결제
- OP202606과 일치: 장바구니쿠폰은 고객별고정가 적용 상품에도 정상 적용

### 첫구매딜 (이탈고객 해당 없음)
이탈고객은 구매이력이 있으므로 첫구매딜 대상이 아님 → 별도 처리 불필요
