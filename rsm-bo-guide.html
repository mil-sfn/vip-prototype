이탈고객 특정상품
가격고정(계약가) 관리

이탈 위험 고객에게 특정 상품의 가격을 고정하여 재구매를 유도하는 기능입니다. 이 가이드는 BO 운영자가 고객별·상품별 고정가를 등록·관리할 때 꼭 알아야 할 내용을 정리했습니다.

배포일 2026-07-22 대상: BO 운영자 메뉴: 고객별고정가 관리
Overview

기능 개요 & 화면 구성

고정가는 고객 × 상품 단위로 등록합니다. 등록된 고정가는 고객향 상품목록·상품상세·장바구니·주문서에 단일 가격으로 노출됩니다.

1고객별고정가 관리

고정가 관리 대상 고객의 전체 목록. 상호명/회원번호 검색, [고객 추가], 사용여부 On/Off 관리.

2고객별고정가 상세

특정 고객에 묶인 상품·고정가의 등록·수정·삭제. 목록에서 상호명 클릭 시 진입. 일괄수정 모드 지원.

3상품별고정가 관리

상품 기준으로 전체 고정가를 조회·수정. 특정 상품이 어느 고객에게 어떤 가격으로 걸려있는지 확인.

💡 기본 등록 흐름

① [고객 추가]로 고객·적용기간 등록 → ② 상세 화면에서 상품 검색 후 고정가 입력·저장 → ③ 적용시작일부터 고객향 노출

필수 확인 ①

고정가 적용 조건: 적용기간 + 사용여부

아래 두 조건을 모두 충족해야 고객에게 고정가가 적용됩니다. 하나라도 어긋나면 기존 가격 로직(타임세일·캠페인·정가)으로 노출됩니다.

📅 조건 1 — 적용기간 이내

현재 날짜가 적용 시작일 ~ 종료일 범위 안이어야 합니다.

· 시작일 기본값: D+1(내일)
· 종료일 기본값: 2999-12-31
· 적용기간은 고객정보 영역에서 수정 가능

🔛 조건 2 — 사용여부 Y(On)

고객의 사용여부가 Y 상태여야 합니다.

· Off 전환 시 즉시 기존 가격 로직으로 폴백
· 연결된 유저 타겟 캠페인도 비활성화
· 시간 조건 없이 언제든 전환 가능

🎛 직접 확인해보기 — 조건 조합별 적용 여부
적용기간 이내 O
사용여부 Y
고정가 적용 두 조건 모두 충족 — 고객에게 고정가가 노출·적용됩니다.
필수 확인 ②

가격 적용 정책 — 무엇이 노출되나요?

유효한 고정가가 있으면 기존 가격 우선순위(타임세일 → 캠페인 → 정가)를 전부 무시하고 고정가만 노출합니다. 단, 정가가 더 낮아지면 예외입니다.

📌 원칙 — 할인가가 더 싸도 고정가 노출

타임세일·캠페인 할인가가 고정가보다 낮아도 고정가를 노출합니다. 고정가는 할인 경쟁이 아닌 계약가이기 때문입니다.

📌 예외 — 정가가 고정가와 같거나 낮으면 정가 노출

정가 인하 등으로 정가 ≤ 고정가가 되면 정가를 노출합니다. 고객이 계약가보다 비싸게 사는 상황을 방지합니다. (BO에서는 '고객노출가' 컬럼으로 실제 노출가 확인 가능. 상품 등록·수정 시에도 동일 조건에서 안내 Alert가 뜹니다)

🎛 직접 확인해보기 — 상황별 고객 노출가 (고정가 8,000원 가정)
8,000원 고정가 노출 타임세일가(6,500원)가 고정가보다 낮아도 고정가 8,000원을 노출합니다.
Registration Tips

등록·수정 시 유의사항

고정가 입력과 저장 과정에서 자주 만나는 규칙입니다.

⚖️ 계근상품·묶음상품은 입력 기준이 다릅니다

계근상품은 1kg 기준 단가로, 묶음상품은 낱개 기준으로 고정가를 입력하세요. 묶음상품은 고객향 노출 시 고정가 × 묶음수량으로 자동 계산되어 노출됩니다.

How-to

작업별 가이드

자주 하는 작업 5가지의 단계별 방법입니다. 탭을 눌러 확인하세요.

고객별고정가 관리 → [고객 추가]
고객 검색검색(🔍) 클릭 → 팝업에서 회원코드·상호명으로 검색 → 행 클릭 시 회원번호·상호명 자동완성 (읽기전용)
적용기간 설정시작일 기본값 내일(D+1) / 종료일 기본값 2999-12-31 — 종료일은 시작일보다 이후여야 함
저장필수값 미입력 시 [저장] 버튼 비활성화. 저장 후 상세 화면에서 상품 등록 진행
💡 참고
  • 탈퇴고객은 고정가 관리 고객목록에 조회되지 않아 검색되지 않습니다
  • 적용기간은 등록 후에도 상세 화면 고객정보 영역에서 수정할 수 있습니다
고객별고정가 관리 → 상호명 클릭 → 고객별고정가 상세 → 상품 등록 폼
상품 검색검색(🔍) 클릭 → 상품 검색 팝업(상품타입·계근여부·상품ID·ERP코드·상품명) → 선택 시 상품 정보 자동완성. 첫구매딜 상품은 선택 불가
참고 정보 확인정가·판매가·과세여부·AI쿠폰금액·최종할인가가 읽기전용으로 표시됩니다 (고정가 산정 참고용)
고정가 입력0 초과 정수만 입력하면 됩니다 (정가·판매가 상한 제약 폐지, 2026-07-07). 정가 ≤ 고정가로 입력 시 안내 Alert가 뜨지만 저장은 차단되지 않습니다: "고정가가 정가와 같거나 크면 고객에게 정가로 노출됩니다."
저장 확인확인 다이얼로그 후 저장 완료 → 상품 목록 갱신
"저장하시겠습니까? 적용시작일(M월D일)부터 입력하신 고정가가 적용됩니다."
⚖️ 계근상품 · 묶음상품 입력 기준

계근상품은 1kg 기준 단가, 묶음상품은 낱개 기준으로 입력하세요. 묶음상품은 고객향에서 고정가 × 묶음수량으로 자동 노출됩니다.

💡 참고
  • 이미 등록된 상품을 선택하면 기존 입력값이 조회됩니다 (수정 모드)
  • 기존 상품 클릭 후 [초기화] 버튼을 누르면 폼이 초기화되어 새 상품을 등록할 수 있습니다
고객별고정가 상세 → 상품 목록 → 삭제(🗑)
삭제 클릭상품 목록 각 행의 삭제(🗑) 버튼 클릭 (시각 무관, 언제든 가능)
확인 다이얼로그"{상품명} 상품을 삭제하시겠습니까? 삭제가 완료되는 즉시 기존 가격이 노출됩니다."
삭제 완료목록에서 행 제거 → 해당 상품의 캠페인 비활성화, 즉시 기존 가격 로직 적용
⚠️ 주의
  • 삭제는 상품 단위입니다. 고객 전체의 고정가를 중단하려면 삭제가 아닌 사용여부 Off로 처리하세요
  • 삭제 즉시 고객에게 기존 가격(타임세일·캠페인·정가)이 노출됩니다
고객별고정가 상세 → 상품 목록에서 행 클릭 → 수정 폼
상품 선택상품 목록에서 수정할 행 클릭 → 폼에 상품ID·ERP코드·상품명·과세여부·정가·판매가·고정가 표시 (최종할인가·AI쿠폰금액은 수정 시 미노출)
고정가·기타 수정등록과 동일하게 0 초과 정수만 확인합니다 (상한 제약 폐지). 정가 ≤ 고정가로 수정하면 안내 Alert가 뜨지만 저장은 차단되지 않습니다.
저장 확인확인 다이얼로그 후 저장 → 목록 갱신
💡 여러 건을 한 번에 수정하려면 — [일괄수정] 모드
  • 상품 목록 상단 [일괄수정] 클릭 → 목록에서 고객별고정가·기타만 직접 입력 가능, 변경된 행은 구분 표시
  • [저장]은 현재 페이지의 변경 건만 저장됩니다 — 다른 페이지는 별도 저장 필요
  • 변경사항이 있는 채로 페이지 이동 시 확인 다이얼로그: "N건의 변경사항이 저장되지 않습니다. 이동하시겠습니까?"
상품별고정가 관리 탭 → [일괄수정]
상품 기준 조회상품ID·상품ERP코드·상품명으로 검색 — 특정 상품이 어느 고객에게 어떤 가격으로 걸려있는지 한눈에 확인 (상호·회원번호 컬럼 표시)
일괄수정 모드 진입목록 상단 [일괄수정] 클릭 → 고객별고정가·기타 필드가 입력 가능 상태로 전환, 변경 행 구분 표시
값 입력유효성은 단건 수정과 동일 — 0 초과 정수만 확인 (상한 제약 폐지). 정가 ≤ 고정가 입력 시 안내 Alert가 뜨지만 저장은 차단되지 않습니다.
저장현재 페이지의 변경 건만 저장 후 일괄수정 모드 해제
"N건 고객별고정가가 일괄 수정되었습니다."
⚠️ 주의
  • 페이지당 25건 표시 — 여러 페이지에 걸친 수정은 페이지마다 저장해야 합니다
  • 변경사항이 있는 채로 페이지 이동 시 미저장 확인 다이얼로그가 뜹니다 (확인 시 변경사항 유실)
  • 동일 상품이라도 고정가는 고객별로 각각 관리됩니다 — 행 단위로 입력하세요
